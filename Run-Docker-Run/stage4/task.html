Description

In the previous stage, you built a Docker image for your Kotlin-based Spring Boot web application. Now, it's time to create and run a container based on this image. Containers are running instances of images and enable you to deploy your application in a consistent and portable way.

In this stage, you will create and run a container from the hyper-web-app image. The container should run in detached mode and map port 8080 on the container to port 8080 on your host machine. Once the container is running, the application should be accessible through your browser or any HTTP client.

Objectives

  • Create a container from the hyper-web-app image.
  • Run the container in detached mode.
  • Map container port 8080 to host port 8080.
