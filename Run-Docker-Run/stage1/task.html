Description

Docker is an essential tool in modern software development. It enablies developers to create portable, scalable, and efficient applications. In this project, you will containerize a simple Kotlin-based Spring Boot web application. This app greets users with a personalized message and displays fun facts about Kotlin, Docker, and Spring Boot when a button is clicked. Go ahead and download this awesome application. In this stage, you will lay the foundation for future stages. You'll create a Dockerfile and .dockerignore file, and pull the necessary base images for JVM-based applications. The Dockerfile is a key component in Docker workflows, defining the steps required to build and package an application into a container. The .dockerignore file is equally important, as it helps optimize your build process by excluding unnecessary files and directories from the Docker build context. You can add entries such as:

# Gradle build files and directories
.gradle
build/

# IDE-specific files
.idea/
*.iml

# Operating system-specific files
.DS_Store
Thumbs.db

# Logs and temporary files
*.log
tmp/

# The Dockerfile and .dockerignore themselves
Dockerfile
.dockerignore

By completing this stage, you will learn how to set up a Dockerfile, configure a .dockerignore file, and pull base images.

While the eclipse-temurin images are recommended, you are free to experiment with other JVM-based images such as amazoncorretto or openjdk.

Objectives

  1. Download and extract the Project Files:

    • Download and extract the provided Spring Boot project into a separate folder to avoid conflicts with the project files.
    • The project folder contains the necessary application files for a simple Spring Boot application.

  2. Create the Dockerfile:

    • In the root directory of the extracted application, create a file named Dockerfile (no extension);
    • Add two instructions in the Dockerfile which specify the base images required for building and packaging the application:
      • A build stage that uses a JVM-based image with the JDK to compile the application such as eclipse-temurin:21-jdk;
      • A packaging stage that uses a JVM-based image with the JRE to run the application such as eclipse-temurin:21-jre;
      • You can use any JVM-based base image such as eclipse-temurin, amazoncorretto, or openjdk;

  3. Create the .dockerignore file:

    • In the root directory of the project, create a file named .dockerignore (no extension);
    • Add files to be excluded from the Docker build context;
  4. Pull the images:
    • Use the appropriate command to pull the base images from Dockerhub;
    • Verify that the images were downloaded and are available locally.

By completing this stage, you will have a functional Dockerfile and the necessary images locally. This serves as the foundation for building and running the Spring Boot application in the next stages.

To do this project docker platform needs to be installed on your system. You can install Docker Desktop which is an easy-to-install application for your Mac, Windows, or Linux environment that enables you to build and share containerized applications and microservices.

On some Linux distributions, you may need to add users to the docker group to pass the tests. You can do this by running the following command:

sudo usermod -aG docker $USER
